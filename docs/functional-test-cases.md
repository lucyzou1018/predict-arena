# Predict Arena - 功能测试用例

> **状态标识说明：**
> - **[已自动化]** = 已被合约自动化测试覆盖（`Comprehensive.test.js`），全部通过
> - **[需手动]** = 需要手动测试（钱包交互 / 多浏览器 / 服务端逻辑）
> - **[部分覆盖]** = 合约链上逻辑已测试，但完整前后端流程需手动验证

## 测试环境

- 网络：Base Sepolia
- 入场费：1 USDC
- 房间有效期：5 分钟
- 支付超时：90 秒
- 匹配超时：15 秒
- 预测阶段：60 秒（deadline = 55 秒）
- 结算延迟：30 秒
- 退款宽限期：300 秒（5 分钟）
- 手续费率：5%

| ID | 状态 | 场景 | 操作步骤 | 预期结果 |
|----|------|------|----------|----------|

| PY-02 | [部分覆盖] | 加入者支付入场费 | 房间满员后点击"Pay 1 USDC" | 同 PY-01 |
| PY-03 | [部分覆盖] | 匹配支付入场费 | 匹配成功后点击"Pay 1 USDC" | 同房间支付流程 |
| PY-04 | [需手动] | USDC 已授权 | 额度充足 | 跳过 approve，直接调用 `payForGame` |
| PY-05 | [需手动] | USDC 需授权 | 额度不足 | 先 approve 交易，再 `payForGame` 交易（两次钱包确认） |
